01Research Philosophy

Engineering intelligence from first principles.

I don't start from frameworks. I start from the primitive — the stateless model call, the control loop, the memory that has to survive a context window — and build upward only once each layer is understood and proven.

Intelligence, to me, isn't a single algorithm but the cooperation of many specialized systems: memory, learning, planning, attention, motivation, uncertainty. My work is making those systems legible — to the machine, and to the people reading it.

"What cannot be measured should not be claimed."

The operating principle behind everything I build

02Engineering Principles
P-01

Measurement Before Intelligence

Evidence precedes every intelligence claim. Observable signals are collected before higher-level reasoning is introduced.

P-02

First-Principles Engineering

Hand-roll the primitives — the agent loop, memory, retrieval — before reaching for a framework. Understand why each part exists.

P-03

Test-Driven Development

Every subsystem is proven by deterministic tests before integration — 58 passing tests, not assumptions.

P-04

Modular Architecture

Independent cognitive subsystems, each with one responsibility — replaceable without rewriting the whole.

P-05

Explainability

Internal reasoning stays visible. Every decision is traceable rather than hidden inside a black box.

P-06

Validation Before Complexity

Validate individual intelligence before coordinating collective intelligence. Complexity is earned, never assumed.

05Flagship III Research Frontier

Velith

A grounded, verification-first engineering intelligence — built so competence can only come from what an external verifier will admit. Near-term: a general engineering loop. North star: manufacturing intelligence.

View Repository ↗
Core thesis

Truth is exogenous. No claim is trusted because a model produced it fluently — only when an external verifier admits it. Velith makes competence earned, not asserted.

The falsifiable experiment

One question V0.1 exists to answer: can a grounded agent measurably improve its held-out performance through verified experience? The design is differential — only the memory write-filter changes between arms.

A0Cold — no memorybaseline
A1Unfiltered — every attempt keptRAG control
A2Verified — only what passedTREATMENT
A3Anti-grounding — belief, never runself-confidence
A4Success-only — isolates failure-learningablation

Decisive criterion: A2 must strictly beat A1 — otherwise grounding adds nothing over retrieval.

State-centric by design

A persistent, typed, provenance-tracked state is the substrate. Generators, verifiers, planners, and learners are typed transformations over it. LLMs are components, not the architecture.

Artificial intelligence drew me in through a single question: how does intelligence actually emerge? I'm less interested in using AI to automate tasks than in understanding how machines learn, how memory shapes decisions, how reasoning develops, and how multiple cognitive systems cooperate.

So rather than building conventional software, I build systems that make intelligence visible. MiniFlyWire, Noetica, and Velith trace one progression — making intelligence visible, grounding it in evidence, then holding it to an external verifier. For me, engineering is a way to study intelligence — not just to implement algorithms.

Science fiction first sparked my curiosity; engineering turned it into purpose. I believe AI should strengthen human capability rather than replace it — helping people learn, think, create, and solve problems we can't yet solve. Every project I build is an attempt to make intelligent systems more useful, more understandable, and more aligned with the people who use them.

I don't see AI as the destination. I see it as our most powerful tool for understanding ourselves — and for solving problems today's world cannot yet solve.
